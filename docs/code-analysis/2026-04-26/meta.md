# Suggested agent-instruction additions (META-1)

Drafts per `synthesis.md` §9. Each rule would have prevented ≥3 recurrences of a finding.

- **Test sleeps must cite the missing completion signal** — When a test uses `setTimeout` / `bun:sleep` / `page.waitForTimeout` to bridge async work, the comment above must name the event/promise/observer/lifecycle hook that does NOT exist for this case (e.g., "tmux IPC has no JS-observable for `%session-renamed` delivery"). PR review (or pre-commit grep) rejects sleeps without the comment. Prevents: ~22 TEST-11 / CONC-7 occurrences this run across `tests/unit/server/ws-handle-connection.test.ts` (8+ sites), `tests/e2e/control-mode-window-size.spec.ts:20`, `tests/e2e/control-mode-notifications.spec.ts:30`, `tests/e2e/title.test.ts:31`, `tests/e2e/keyboard.test.ts:25`, `tests/e2e/menu-session-switch-content.spec.ts:96`, plus the production-side `src/server/http.ts:620` shutdown sleep. Rationale: this is the dominant theme in the test suite this run; an explicit comment converts every sleep into a reviewable artifact instead of an invisible flake risk.

- **Every release-artifact path needs a post-package smoke test** — AGENTS.md references `verify-vendor-xterm.ts` as a single example; generalize as a project rule: every artifact users download (Bun `--compile` binary, Electrobun bundle, future Docker/etc.) must have a post-package script that exercises at least one runtime contract — binary executes; sentinel responds; bundle entrypoint loads. Add this as a section in AGENTS.md `## Releases` next to the existing fuzz protocol. Prevents: CI-5 / BUILD-4 recurrences across the v1.8.0 bunfs/embedded-tmux precedent (CHANGELOG.md:63), the current `verify-vendor-xterm` xterm-only coverage gap, and the `tls.test.ts` source-mode-only test. Rationale: the v1.8.0 incident is documented as "regressed at least five times"; adding the rule turns the existing single-artifact pattern into a generalized policy that future contributors can follow without rediscovering it.

- **Document-renamed-files sweep** — When renaming a top-level instruction file, symbol, or removing a feature, run `rg <old-name>` across the repo and update every reference in the same commit. Add as a checklist line to AGENTS.md `## Releases` (or a new `## Renames and removals` section) so it's part of the pre-tag protocol. Prevents: 7+ stale-reference findings this run — README.md:181,194 (`CLAUDE.md` after the AGENTS.md rename), AGENTS.md:316,362 (`#btn-session-plus` "unwired" after wire-up), `tmux-web-dev:6` (`ghostty.js` after renderer consolidation), `tmux-web-dev.service:8` (`--terminal xterm` after flag removal), `package.json:7` ("multiple terminal backends" after multi-backend collapse), `tests/fuzz/README.md:18` (`CLAUDE.md`). Rationale: a 737-commits-in-90-days solo repo has no team-driven cross-tree refactor, so renames need a deliberate sweep step or the residue accumulates.
